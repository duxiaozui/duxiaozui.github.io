SpringBoot框架

第一章 常见面试题

【1】什么是 Spring Boot？

多年来，随着新功能的增加，spring 变得越来越复杂。只需访问https://spring.io/projects 页面，我们就会看到可以在我们的应用程序中使用的所有 Spring 项目的不同功能。如果必须启动一个新的 Spring 项目，我们必须添加构建路径或添加 Maven 依赖关系，配置应用程序服务器，添加 spring 配置。因此，开始一个新的 spring 项目需要很多努力，因为我们现在必须从头开始做所有事情。Spring Boot 是解决这个问题的方法。Spring Boot 已经建立在现有 spring 框架之上。使用 spring 启动，我们避免了之前我们必须做的所有样板代码和配置。因此，Spring Boot 可以帮助我们以最少的工作量，更加健壮地使用现有的 Spring功能。

【2】Spring Boot 有哪些优点？

Spring Boot 的优点有：

  1. 减少开发，测试时间。
  2. 使用 JavaConfig 有助于避免使用 XML。
  3. 避免大量的 Maven 导入和各种版本冲突。
  4. 提供意见发展方法。
  5. 提供默认值快速开始开发。
  6. 没有单独的 Web 服务器需要。这意味着你不再需要启动 Tomcat，Glassfish或其他任何东西。
  7. 需要更少的配置 因为没有 web.xml 文件。只需添加用@ Configuration 注释的类，然后添加用@Bean 注释的方法，Spring 将自动加载对象并像以前一样对其进行管理。您甚至可以将@Autowired 添加到 bean 方法中，以使 Spring 自动装入需要的依赖关系中。

【4】Spring Boot 中的监视器是什么？

Spring boot actuator 是 spring 启动框架中的重要功能之一。Spring boot 监视器可帮助您访问生产环境中正在运行的应用程序的当前状态。有几个指标必须在生产环境中进行检查和监控。即使一些外部应用程序可能正在使用这些服务来向相关人员触发警报消息。监视器模块公开了一组可直接作为 HTTP URL 访问的REST 端点来检查状态。

【5】如何在 Spring Boot 中禁用 Actuator 端点安全性？

默认情况下，所有敏感的 HTTP 端点都是安全的，只有具有 ACTUATOR 角色的用户才能访问它们。安全性是使用标准的 HttpServletRequest.isUserInRole 方法实施的。 我们可以使用来禁用安全性。只有在执行机构端点在防火墙后访问时，才建议禁用安全性。

【10】什么是 Spring Profiles？

【14】什么是 CSRF 攻击？

CSRF 代表跨站请求伪造。这是一种攻击，迫使最终用户在当前通过身份验证的Web 应用程序上执行不需要的操作。CSRF 攻击专门针对状态改变请求，而不是数据窃取，因为攻击者无法查看对伪造请求的响应。

【15】SpringBoot的底层原理？

答：SpringBoot工作原理主要由三个方面构成的，第一个是pom文件继承父工程，父工程将常见启动器做了版本控制。第二个是场景启动器，场景启动器就是maven工程，对应用场景进行封装打包。第三个是启动类，启动类@SpringBootApplication底层注解做了三件事，启动类本身是配置类，启动类加载当前类所在包下的所有注解，启动类按照条件加载第三方场景启动配置类，例如很多都是有class文件就加载启动类，就意味着添加依赖，自动注入场景启动器的对象。

【16】跨域？

答：跨域指的是：从一个域名的网页去请求另一个域名的资源时，域名、端口、协议任一不同，都是跨域。跨域是由浏览器的同源策略造成的，是浏览器施加的安全限制。a页面想获取b页面资源，如果a、b页面的协议、域名、端口、子域名不同，所进行的访问行动都是跨域的

【19】说一下JWT登录的执行流程？

答：Jwt登录流程如下：

  1. 客户端使用用户名和密码请求登录
  2. 服务端收到请求，去验证用户名与密码
  3. 验证成功后，服务器颁发一个Token,再把这个Token发送给客户端（一般情况下，我们会把用户信息存到redis中，存储的key是工程：业务：用户id,value存user转json后的用户信息）
  4. 客户端收到Token以后可以把它存储起来
  5. 客户端每次向服务器端请求资源的时候需要带着服务端颁发的Token
  6. 服务端收到请求，然后去验证客户端请求里面带着的Token,如果验证成功，就向客户端返回请求的数据。（这里可以通过过滤器或拦截器实现，如果用springSecurity框架实现，那么必须用过滤器实现，并将过滤器设置到SpringSecurity的配置类中，保证在最前面执行）

【20】说一下你通过SpringSecurity如何完成登录和授权的。

答：完成登录功能如下：

  1. 获得客户端的用户名和密码，将用户名和密码封装成UsernamePasswordAuthenticationToken，然后将它交给authenticationManager认证，认证返回Authentication对象

  2. 判断认证对象是否为空，为空表示认证失败，不为空认证成功，认证成功做两件事

    1. 将用户信息存入redis
    2. 根据用户信息生成token返回客户端

  3. 此后的每次请求都要携带token，被认证过滤器拦截，认证过滤器获取token，解析token，解析失败返回认证失败，解析成功刷新token，刷新redis,将用户信息封装到UsernamePasswordAuthenticationToken对象。最后通过 SecurityContextHolder.getContext().setAuthentication(usernamePasswordAuthenticationToken);

    表示用户认证通过。

  4. 前面的功能需要配置SpringSecurity配置类的前提下使用

授权功能如下：

  1. 先用户分配角色功能
  2. 在完成角色分配菜单功能
  3. 客户端访问后端接口的时候查询用户拥有的角色，角色拥有的权限，去和方法上的权限注解去比对，比对成功响应数据，失败返回权限不足。
  4. 常见权限注解是 @PreAuthorize(“hasAuthority(‘test’)”)
  5. 使用权限注解需要在配置类上开启开关：@EnableWebSecurity //开启springSecurity框架
    @EnableGlobalMethodSecurity(prePostEnabled = true)//开启权限注解开发
